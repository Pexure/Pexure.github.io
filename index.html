•P操作就是从池子里取走一把锁。如果取成功，线程继续执行，如果取失败，就必须等待。
•V操作就是把一把锁放进池子里。这时候如果有线程在等锁，那个线程可以取走这把锁执行。

notes:
1)P/V是原子操作
2)仅有一个手环(count=1) = 互斥锁
3)P = wait; V = signal
4)因为计数器的存在，不会发生signal“丢失”

虽说二分的原理非常简单，代码也不长，但是我却难以做到又快又稳地写对…(算法竞赛，菜是原罪)

先码一下lower_bound()和upper_bound(),begin和end是左闭右开[begin,end)

lower_bound(begin,end,num)：从数组的begin位置到end-1位置二分查找第一个大于或等于num的数字，找到返回该数字的地址，不存在则返回end。通过返回的地址减去起始地址begin,得到找到数字在数组中的下标。

upper_bound(begin,end,num)：从数组的begin位置到end-1位置二分查找第一个大于num的数字，找到返回该数字的地址，不存在则返回end。通过返回的地址减去起始地址begin,得到找到数字在数组中的下标。

在从大到小的排序数组中，重载lower_bound()和upper_bound()

lower_bound(begin,end,num,greater() ):从数组的begin位置到end-1位置二分查找第一个小于或等于num的数字，找到返回该数字的地址，不存在则返回end。通过返回的地址减去起始地址begin,得到找到数字在数组中的下标。

upper_bound(begin,end,num,greater() ):从数组的begin位置到end-1位置二分查找第一个小于num的数字，找到返回该数字的地址，不存在则返回end。通过返回的地址减去起始地址begin,得到找到数字在数组中的下标。

为了精准理解和写对二分（不光是查找），还是得自己写一个

int _lower_bound(int l,int r,int v){
while (l<r){
int m=(l+r)>>1;//默认正数，不溢出
if (a[m]>=v) r=m;//若满足 往一个方向缩小查找范围
else l=m+1;//否则向另一个方向缩小
}
return l;
//终止时，一定l==r，不存在l>r
}
分析一哈：这里和标准库一样，采用左闭右开[l,r)
即查找区间[l,r)，但是返回区间是[l,r] （如果a[r-1]都比v小，则返回r）
a[m]和v之间的关系可能带来的影响如下：
1.a[m]>=v 已经找到一个，但[l,m)之间可能还有，返回区间变为[l,m]
2.a[m]<v [l,m]都不可能了，返回区间变为[m+1,r]

int _upper_bound(int l,int r,int v){
while (l<r){
int m=(r+l)>>1;
if (a[m]>v) r=m;
else l=m+1;
}
return l;
}

Time Limit: 15 Sec Memory Limit: 162 MB

Description
某加工厂有A、B两台机器，来加工的产品可以由其中任何一台机器完成，或者两台机器共同完成。由于受到机器性能和产品特性的限制，不同的机器加工同一产品所需的时间会不同，若同时由两台机器共同进行加工，所完成任务又会不同。某一天，加工厂接到n个产品加工的任务，每个任务的工作量不尽一样。你的任务就是：已知每个任务在A机器上加工所需的时间t1, B机器上加工所需的时间t2及由两台机器共同加工所需的时间t3，请你合理安排任务的调度顺序，使完成所有n个任务的总时间最少。
