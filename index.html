二分查找

发表于 2019-03-01

虽说二分的原理非常简单，代码也不长，但是我却难以做到又快又稳地写对…(算法竞赛，菜是原罪)

先码一下lower_bound()和upper_bound(),begin和end是左闭右开[begin,end)

lower_bound(begin,end,num)：从数组的begin位置到end-1位置二分查找第一个大于或等于num的数字，找到返回该数字的地址，不存在则返回end。通过返回的地址减去起始地址begin,得到找到数字在数组中的下标。

upper_bound(begin,end,num)：从数组的begin位置到end-1位置二分查找第一个大于num的数字，找到返回该数字的地址，不存在则返回end。通过返回的地址减去起始地址begin,得到找到数字在数组中的下标。

在从大到小的排序数组中，重载lower_bound()和upper_bound()

lower_bound(begin,end,num,greater() ):从数组的begin位置到end-1位置二分查找第一个小于或等于num的数字，找到返回该数字的地址，不存在则返回end。通过返回的地址减去起始地址begin,得到找到数字在数组中的下标。

upper_bound(begin,end,num,greater() ):从数组的begin位置到end-1位置二分查找第一个小于num的数字，找到返回该数字的地址，不存在则返回end。通过返回的地址减去起始地址begin,得到找到数字在数组中的下标。

为了精准理解和写对二分（不光是查找），还是得自己写一个

int _lower_bound(int l,int r,int v){
while (l<r){
int m=(l+r)>>1;//默认正数，不溢出
if (a[m]>=v) r=m;//若满足 往一个方向缩小查找范围
else l=m+1;//否则向另一个方向缩小
}
return l;
//终止时，一定l==r，不存在l>r
}
分析一哈：这里和标准库一样，采用左闭右开[l,r)
即查找区间[l,r)，但是返回区间是[l,r] （如果a[r-1]都比v小，则返回r）
a[m]和v之间的关系可能带来的影响如下：
1.a[m]>=v 已经找到一个，但[l,m)之间可能还有，返回区间变为[l,m]
2.a[m]<v [l,m]都不可能了，返回区间变为[m+1,r]

int _upper_bound(int l,int r,int v){
while (l<r){
int m=(r+l)>>1;
if (a[m]>v) r=m;
else l=m+1;
}
return l;
}

1222: [HNOI2001]产品加工
Time Limit: 15 Sec Memory Limit: 162 MB

Description
某加工厂有A、B两台机器，来加工的产品可以由其中任何一台机器完成，或者两台机器共同完成。由于受到机器性能和产品特性的限制，不同的机器加工同一产品所需的时间会不同，若同时由两台机器共同进行加工，所完成任务又会不同。某一天，加工厂接到n个产品加工的任务，每个任务的工作量不尽一样。你的任务就是：已知每个任务在A机器上加工所需的时间t1, B机器上加工所需的时间t2及由两台机器共同加工所需的时间t3，请你合理安排任务的调度顺序，使完成所有n个任务的总时间最少。

输入共n+1行第1行为 n。 n是任务总数（1≤n≤6000）第i+1行为3个[0，5]之间的非负整数t1,t2,t3，分别表示第i个任务在A机器上加工、B机器上加工、两台机器共同加工所需要的时间。如果所给的时间t1或t2为0表示任务不能在该台机器上加工，如果t3为0表示任务不能同时由两台机器加工。

经典的滚动数组: 一维 f[M]
0/1背包 （第二维从大到小）
完全背包 （第二维从小到大）

常见滚动数组： 一维*2 f[2][M]

首先不管怎样，强烈建议先写出非滚动数组的版本，再改成滚动数组，因为这涉及初始化和更新的很多细节

若不滚动时需要初始化为INF，需要考虑滚动后的值已经不是INF，参见bzoj 1222
若是涉及计数（累加，+=），则f[cur][j]在循环结束时需要设为0，有道这样的题，整理时再补充

Solution:
状态表示：
首先有关时间和二维平面的dp最普通的状态表示为f[x][y][t]：t时刻在点(x,y)的最优值
此题总时间T应该很大，上述表示显然会T。题目用了区间表示时间，已经明显地暗示了状态f[x][y][k]表示第k段时间的结束时刻，在点(x,y)的最优值

状态转移：
传统的状态转移一次最多走一格，这里用区间表示则可以一次转移走多格。
以向右移动为例，f[x][y][k]=max{f[x][y’][k-1]+y-y’}=y+max{f[x][y’][k-1]-y’},其中y-y’<=t[k].end-t[k].start+1
这样就变成了经典的滑窗最大值问题，可用单调队列优化，在O(n)时间内解决。

关于单调队列，以后就都用[l,r)表示吧，左闭右开，初始化l=r=0

Code:

#include<bits/stdc++.h>
using namespace std;
const int mod=998244353;
//const int mod=1e9+7;
typedef long long ll;
const int N=1000;

#define INF 0x3f3f3f3f

#define pb push_back

#define clr(x) memset(x,0,sizeof(x))

#define mp make_pair
const int dx[4]={-1,1,0,0};
const int dy[4]={0,0,-1,1};
int T,n,kase,qry,ql,qr,op,m,k,cur,ans;
char g[N][N];
pair<int,int> q[N];
int f[2][N][N];

void solve(int x,int y,int w,int len,int dir){
int l=0,r=0,tmp;
for (int i=1;i<=w;i++){
if (g[x][y]==’x’){
f[cur][x][y]=-INF;
l=r=0;
}
else{
tmp=f[!cur][x][y]-i;
while (l<r && q[r-1].second<=tmp) r–;
q[r++]=mp(i,tmp);
while (llen) l++;
f[cur][x][y]=q[l].second+i;
//ans=max(ans,f[cur][x][y]);
}
x+=dx[dir];
y+=dy[dir];
}
}

int main(){
int x,y;
scanf(“%d%d%d%d%d”,&n,&m,&x,&y,&k);
for (int i=1;i<=n;i++) scanf(“%s”,g[i]+1);
memset(f,-INF,sizeof(f));
f[0][x][y]=0;
for (int a,b,c,i=1;i<=k;i++){
cur=i&1;
scanf(“%d%d%d”,&a,&b,&c);
if (c==1) for (int j=1;j<=m;j++) solve(n,j,n,b-a+1,c-1);
if (c==2) for (int j=1;j<=m;j++) solve(1,j,n,b-a+1,c-1);
if (c==3) for (int j=1;j<=n;j++) solve(j,m,m,b-a+1,c-1);
if (c==4) for (int j=1;j<=n;j++) solve(j,1,m,b-a+1,c-1);
}
for (int i=1;i<=n;i++){
for (int j=1;j<=m;j++) {
ans=max(ans,f[cur][i][j]);
//printf(“%d “,f[cur][i][j]);
}
//cout<<endl;
}
printf(“%d”,ans);
return 0;
}
