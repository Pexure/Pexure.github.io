关于滚动数组

发表于 2019-02-27

经典的滚动数组: 一维 f[M]
0/1背包 （第二维从大到小）
完全背包 （第二维从小到大）

常见滚动数组： 一维*2 f[2][M]

首先不管怎样，强烈建议先写出非滚动数组的版本，再改成滚动数组，因为这涉及初始化和更新的很多细节

若不滚动时需要初始化为INF，需要考虑滚动后的值已经不是INF，参见bzoj 1222
若是涉及计数（累加，+=），则f[cur][j]在循环结束时需要设为0，有道这样的题，整理时再补充

滚动数组有时更快是因为cache的工作原理

1499: [NOI2005]瑰丽华尔兹
Time Limit: 3 Sec Memory Limit: 64 MB

Description
(你跳过华尔兹吗？当音乐响起，当你随着旋律滑动舞步，是不是有一种漫步仙境的惬意？ 众所周知，跳华尔兹时，最重要的是有好的音乐。但是很少有几个人知道，世界上最伟大的钢琴家一生都漂泊在大海上，他的名字叫丹尼•布德曼•T.D.•柠檬•1900，朋友们都叫他1900。 1900在20世纪的第一年出生在往返于欧美的邮轮弗吉尼亚号上，很不幸他刚出生就被抛弃了，成了孤儿。1900孤独的成长在弗吉尼亚号上，从未离开过这个摇晃的世界。也许是对他命运的补偿，上帝派可爱的小天使艾米丽照顾他。 可能是天使的点化，1900拥有不可思议的钢琴天赋：从未有人教，从没看过乐谱，但他却能凭着自己的感觉弹出最沁人心脾的旋律。当1900的音乐获得邮轮上所有人的欢迎时，他才8岁，而此时的他已经乘着海轮往返欧美大陆50余次了。 虽说是钢琴奇才，但1900还是个孩子，他有着和一般男孩一样的好奇和调皮，只不过更多一层浪漫的色彩罢了： 这是一个风雨交加的夜晚，海风卷起层层巨浪拍打着弗吉尼亚号，邮轮随着巨浪剧烈的摇摆。船上的新萨克斯手马克斯•托尼晕船了，1900招呼托尼和他一起坐上舞厅里的钢琴，然后松开了固定钢琴的闸，于是，钢琴随着海轮的倾斜滑动起来。准确的说，我们的主角1900、钢琴、邮轮随着1900的旋律一起跳起了华尔兹，随着“嘣嚓嚓”的节奏，托尼的晕船症也奇迹般的消失了。后来托尼在回忆录上写道： 大海摇晃着我们 使我们转来转去 快速的掠过灯和家具 我意识到我们正在和大海一起跳舞 真是完美而疯狂的舞者 晚上在金色的地板上快乐的跳着华尔兹是不是很惬意呢？也许，我们忘记了一个人，那就是艾米丽，她可没闲着：她必须在适当的时候施展魔法帮助1900，不让钢琴碰上舞厅里的家具。)
不妨认为舞厅是一个N行M列的矩阵，矩阵中的某些方格上堆放了一些家具，其他的则是空地。钢琴可以在空地上滑动，但不能撞上家具或滑出舞厅，否则会损坏钢琴和家具，引来难缠的船长。 每个时刻，钢琴都会随着船体倾斜的方向向相邻的方格滑动一格，相邻的方格可以是向东、向西、向南或向北的。而艾米丽可以选择施魔法或不施魔法：如果不施魔法，则钢琴会滑动；如果施魔法，则钢琴会原地不动。 艾米丽是个天使，她知道每段时间的船体的倾斜情况。她想使钢琴在舞厅里滑行路程尽量长，这样1900会非常高兴，同时也有利于治疗托尼的晕船。但艾米丽还太小，不会算，所以希望你能帮助她。

Input
输入文件的第一行包含5个数N, M, x, y和K。N和M描述舞厅的大小，x和y为钢琴的初始位置（x行y列）；我们对船体倾斜情况是按时间的区间来描述的，且从1开始计量时间，比如“在[1, 3]时间里向东倾斜，[4, 5]时间里向北倾斜”，因此这里的K表示区间的数目。 以下N行，每行M个字符，描述舞厅里的家具。第i行第j列的字符若为‘ . ’，则表示该位置是空地；若为‘ x ’，则表示有家具。 以下K行，顺序描述K个时间区间，格式为：si ti di。表示在时间区间[si, ti]内，船体都是向di方向倾斜的。di为1, 2, 3, 4中的一个，依次表示北、南、西、东（分别对应矩阵中的上、下、左、右）。输入保证区间是连续的，即 s1 = 1 si = ti-1 + 1 (1 < i ≤ K) tK = T

Output
输出文件仅有1行，包含一个整数，表示钢琴滑行的最长距离(即格子数)。
Sample Input
4 5 4 1 3
..xx.
…..
…x.
…..
1 3 4
4 5 1
6 7 3

Sample Output
6

/—————————————————-/

Solution:
状态表示：
首先有关时间和二维平面的dp最普通的状态表示为f[x][y][t]：t时刻在点(x,y)的最优值
此题总时间T应该很大，上述表示显然会T。题目用了区间表示时间，已经明显地暗示了状态f[x][y][k]表示第k段时间的结束时刻，在点(x,y)的最优值

状态转移：
传统的状态转移一次最多走一格，这里用区间表示则可以一次转移走多格。
以向右移动为例，f[x][y][k]=max{f[x][y’][k-1]+y-y’}=y+max{f[x][y’][k-1]-y’},其中y-y’<=t[k].end-t[k].start+1
这样就变成了经典的滑窗最大值问题，可用单调队列优化，在O(n)时间内解决。

关于单调队列，以后就都用[l,r)表示吧，左闭右开，初始化l=r=0

Code:

#include<bits/stdc++.h>
using namespace std;
const int mod=998244353;
//const int mod=1e9+7;
typedef long long ll;
const int N=1000;

#define INF 0x3f3f3f3f

#define pb push_back

#define clr(x) memset(x,0,sizeof(x))

#define mp make_pair
const int dx[4]={-1,1,0,0};
const int dy[4]={0,0,-1,1};
int T,n,kase,qry,ql,qr,op,m,k,cur,ans;
char g[N][N];
pair<int,int> q[N];
int f[2][N][N];

void solve(int x,int y,int w,int len,int dir){
int l=0,r=0,tmp;
for (int i=1;i<=w;i++){
if (g[x][y]==’x’){
f[cur][x][y]=-INF;
l=r=0;
}
else{
tmp=f[!cur][x][y]-i;
while (l<r && q[r-1].second<=tmp) r–;
q[r++]=mp(i,tmp);
while (llen) l++;
f[cur][x][y]=q[l].second+i;
//ans=max(ans,f[cur][x][y]);
}
x+=dx[dir];
y+=dy[dir];
}
}

int main(){
int x,y;
scanf(“%d%d%d%d%d”,&n,&m,&x,&y,&k);
for (int i=1;i<=n;i++) scanf(“%s”,g[i]+1);
memset(f,-INF,sizeof(f));
f[0][x][y]=0;
for (int a,b,c,i=1;i<=k;i++){
cur=i&1;
scanf(“%d%d%d”,&a,&b,&c);
if (c==1) for (int j=1;j<=m;j++) solve(n,j,n,b-a+1,c-1);
if (c==2) for (int j=1;j<=m;j++) solve(1,j,n,b-a+1,c-1);
if (c==3) for (int j=1;j<=n;j++) solve(j,m,m,b-a+1,c-1);
if (c==4) for (int j=1;j<=n;j++) solve(j,1,m,b-a+1,c-1);
}
for (int i=1;i<=n;i++){
for (int j=1;j<=m;j++) {
ans=max(ans,f[cur][i][j]);
//printf(“%d “,f[cur][i][j]);
}
//cout<<endl;
}
printf(“%d”,ans);
return 0;
}

P1 (15)
Description：
服务器有n个请求等待处理，第i个请求所需时间为ti，尚未处理的请求需要等待，求总等待时间之和最小。

Solution:

P2 (20)
Description:
带权无向连通图G=(V,E,w)，e=(u,v)∈E。
证明：e不属于G的任何最小生成树当且仅当u、v之间存在由权重小于w(e)的边构成的路径。

Solution:
有关生成树证明问题的两个重要性质：Cycle Property和Cut Property
Cycle Property: 环上权重最大的边（唯一）不在任何最小生成树中
Cut Proprety: 将G分为两个非空点集S,T，则跨越S,T的权重最小的边（唯一）在所有最小生成树中
然后xjb搞就行了。

Complement:
Give an O(n+m) alg. to decide whether e=(u,v) is contained in any MST of G
删去所有权重小于w(e)的边，看u,v是否连通即可。（直接出这个就更炸了…

P3 (25)
Description:
无向连通图G=(V,E)，定义 “SCC定向”：如果能对G的每条边确定一个方向，使得定向后的有向图是强连通的，则称G存在SCC定向。
(1)证明G存在SCC定向当且仅当G中没有桥。
(2)假设G中没有桥，请设计一个线性时间算法给出G的一种SCC定向

Solution:
分数全靠这题混…
(1)
‘=>’：由强连通定义，显然。
‘<=’: 即给出一种构造，参见(2)
(2)
DFS即可。
没有桥即G是2连通的，即所有边都在环中。先考虑单独一个环，定向；然后不断加ears即可(离散课件上说是H-path)。ears分closed ears(连接点为割点)和open ears(非割点)。
为什么只要DFS一遍就行?理解关键点在于对于无向图，DFS可以将所有边分类为tree edge和back edge，使图变得具有层次。考虑某一“子树”的根rt，以它儿子为根的所有“子树”中必有点连回rt，否则存在桥。

P4 (20)
Description：
带权无向连通图G
定义”瓶颈长度”： 一条路径的瓶颈长度指路径上最小的边权
定义”瓶颈距离”： 两点间的瓶颈距离指两点间所有路径中的最大瓶颈长度。
设计O(n^3)算法求G中所有点对之间的瓶颈距离。

Solution：
显然Floyd
需要注意的是，有没有负边，有没有负环都是一样的（两种情况都讨论下就知道了）

P5 (10)
Description：
(1)写出Menger定理
(2)使用Menger定理证明：如果G是3正则图，则κ(G)=λ(G)

Solution:
(1)
u,v是k连通的当且仅当u,v之间存在k条除端点外不相交路径（大概
(2)
已经有κ(G)<=λ(G)<=δ(G)，只要证κ(G)=>λ(G)
主要就是任意点对u,v之间必然有3条不相交路径，因为如果有相交路径，则交点度数>=4

P6 (5)
Description：
证明或证伪：对于正整数n，在4*n的棋盘上，不存在“马踏棋盘回路”(马走的哈密尔顿回路)

Solution：
显然证明。不过归纳是归纳不出来的，这辈子都…
按棋盘坐标和奇偶进行黑白染色。将棋盘中间两行（共4行）的黑点全删了，再根据判断哈密尔顿回路必要性的那个定理即可得证。因为删了中间两行黑点，则第1,4行的白点全变成孤立点，且数量和删掉的黑点相同，剩下的点至少是一个连通块。

P7 (5)
Description：
考虑无向连通图G上的双人游戏，两位玩家交替选点。
规则：第一个人先任选一个点，接下来每位玩家的选点须满足1.与另一外玩家刚刚选的点相邻2.未被选过（即两位玩家选择的顶点构成一条简单路径）
若无顶点可选，则该玩家输，他的对手赢。
问：当G具有何种性质时，后手玩家有必胜策略，否则先手玩家有必胜策略，并证明。

Solution:
哎现在看到”构成简单路径”马上就想到了增广路。上学期脑(xue)子(de)坏(la)了(ji)。
有完美匹配则后手必胜，否则先手必胜。
先手必胜只需考虑G的最大匹配，然后第一个选那个唯一的非匹配点即可。(必然只有一个非匹配点，否则存在增广路，不是最大匹配。这点日后做相关oj的时候再重新研究一下吧)

P8 (附加题)
Description：
设计一个线性时间算法判断有向图G是否含有有向奇环。

Solution:
先找G的scc（有向环），对每个scc，将边的方向去掉，判二分图（奇环）即可。
